## 2.2. OpenAI兼容格式
在模型接入场景中，很多厂商都会提供 OpenAI 兼容接口，工程中也常被口语化称为 `openapi` 格式。它的核心特点是请求结构统一、迁移成本低、便于在不同模型服务之间切换。最常见的端点是 `POST /v1/chat/completions` 或 `POST /v1/responses`，调用时通常需要在请求头中传入 `Authorization: Bearer <API_KEY>`，请求体使用 JSON 格式。

这种格式的核心入参通常包括：

- `model`: 要调用的模型名称，例如 `gpt-4o`、`deepseek-chat`、`qwen-plus`。
- `messages`: 对话数组，元素包含 `role` 和 `content`。常见角色包括 `system`、`user`、`assistant`。
- `temperature`: 控制输出随机性，值越高越发散，值越低越稳定。
- `max_tokens` 或 `max_completion_tokens`: 控制本次生成的最大输出长度。
- `stream`: 是否使用流式输出，`true` 时一般通过 SSE 持续返回增量内容。
- `tools` 和 `tool_choice`: 用于函数调用或外部工具调用。

一个典型请求体如下：

```json
{
  "model": "gpt-4o",
  "messages": [
    {
      "role": "system",
      "content": "你是一个专业的技术文档助手。"
    },
    {
      "role": "user",
      "content": "请介绍 OpenAI 兼容接口的基本结构。"
    }
  ],
  "temperature": 0.2,
  "max_tokens": 800,
  "stream": false
}
```

典型响应体通常包含以下字段：

- `id`: 本次请求的唯一标识。
- `object`: 对象类型，例如 `chat.completion`。
- `created`: 创建时间戳。
- `model`: 实际返回所使用的模型名称。
- `choices`: 模型输出列表，核心内容一般位于 `choices[0].message.content`。
- `usage`: Token 使用统计，通常包含 `prompt_tokens`、`completion_tokens`、`total_tokens`。

一个典型响应体如下：

```json
{
  "id": "chatcmpl-123",
  "object": "chat.completion",
  "created": 1710000000,
  "model": "gpt-4o",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "OpenAI 兼容接口通常使用统一的 JSON 请求结构。"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 28,
    "completion_tokens": 16,
    "total_tokens": 44
  }
}
```

## 2.3. OpenAI兼容格式示例
使用 OpenAI 兼容格式的优势在于，一套调用结构可以复用于多个模型平台，只需替换 `base_url`、`api_key` 和 `model` 等少量配置。下面给出常见的调用示例。

使用 `curl` 调用示例：

```bash
curl https://api.openai.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "你是一个技术助手。"},
      {"role": "user", "content": "请解释什么是 OpenAI 兼容格式。"}
    ],
    "temperature": 0.2,
    "max_tokens": 300
  }'
```

使用 Python SDK 的兼容调用示例：

```python
from openai import OpenAI

client = OpenAI(
    api_key="your_api_key",
    base_url="https://api.openai.com/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一个技术助手。"},
        {"role": "user", "content": "请解释什么是 OpenAI 兼容格式。"}
    ],
    temperature=0.2,
    max_tokens=300
)

print(response.choices[0].message.content)
```

如果切换到支持 OpenAI 兼容协议的其他厂商，通常只需要改 `base_url` 和 `model`，例如：

```python
from openai import OpenAI

client = OpenAI(
    api_key="your_api_key",
    base_url="https://api-docs.deepseek.com"
)

response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "user", "content": "请总结 OpenAI 兼容接口的优点。"}
    ]
)

print(response.choices[0].message.content)
```
