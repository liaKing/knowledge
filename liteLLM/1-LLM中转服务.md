LLM中转服务

### 02-LLM中转服务
LLM中转服务旨在通过集成LiteLLM，构建一个高效、可扩展且易于管理的LLM代理层。该服务将LiteLLM作为核心流量处理引擎，而Go主服务则退居幕后，专注于管理和计费逻辑，从而实现客户端与LLM服务之间的无缝、安全和成本可控的交互。

## 1. 主要流程
1.1. LiteLLM：负责处理所有大流量并发、流式输出、Token 计算、计费拦截。
1.2. Go 主服务：退居幕后，变成一个“管理中枢”，只负责通过调用 LiteLLM 的 Admin API 为用户生成 Key、充值额度、拉取账单。它完全不参与核心的聊天数据流，极大地降低了 Go 服务的压力。

## 2. 详细流程
### 2.1. 开户/注册 (Go 服务)
用户在您的系统注册，Go 服务调用 LiteLLM 的 `/key/generate` 接口，为该用户生成一个专属的 `sk-xxx` API Key，并设定他的可用预算 (`max_budget`) 和可用模型 (`models`)。

### 2.2. 直连聊天 (客户端直连 LiteLLM)
前端直接拿着这个专属 Key，向 LiteLLM 发起 `/v1/chat/completions` 流式请求。

### 2.3. 拦截与扣费 (LiteLLM 内部)
LiteLLM 会自动校验这个 Key 的有效性，校验预算余额。如果在通话途中或发起前余额耗尽，LiteLLM 会自动拦截并返回错误（不需要 Go 服务干预）。
